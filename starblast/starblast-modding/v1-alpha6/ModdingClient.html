ModdingClient

ModdingClient

The Modding Client Instance.

Constructor

new ModdingClient(options)

Parameters:
Name Type Description
options object options for calling the object. Note that if mod fails to start, all cached data are not erased.
Properties
Name Type Attributes Default Description
cacheECPKey boolean <optional>
 false set to `true` if you want to reuse ECP Key for the next run, `false` otherwise
cacheOptions boolean <optional>
 false set to `true` if you want to reuse request options for the next run, `false` otherwise
cacheEvents boolean <optional>
 false set to `true` if you want to reuse all event handlers for the next run, `false` otherwise
compressWSMessages boolean <optional>
 false To decide whether to compress messages in WebSocket requests or not. `true` will use less bandwith but may also be CPU-intensive, and `false` will do the opposite.
extendedMode boolean <optional>
 false Whether to enable extended listener (which includes extended events and ship properties such as station-related events or ship customization)
